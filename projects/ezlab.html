INTRODUCTION

In order to solve a problem I encountered during my research assistant work, I initiate researches, ideation and prototyping to envision a cross platform system for two distinguished user groups.

​In 2019, after taking SI582 Introduction to Interaction Design and SI649 Information Visualization, I reopened this project based on user research results and refined the prototypes.

EZLab solves the problem of the inefficiency in recruiting experiment volunteers.

It connects researchers and experiment volunteers with matching interests and automatic information exchanging.

It has mobile for volunteers to browse, search and sign up for interested experiments. It has website for researchers to monitor, message volunteers and save more time for research work.

USER RESEARCH

I started this project because of a problem I found in life

Talk to both stakeholders to understand the problems

I contacted 4 student researchers who took part in volunteer recruitment for research projects and 2 experiment volunteers of their ongoing projects. I conducted semi-structured interviews with researchers about the whole process and their feelings in different phases of managing the recruitment. I conducted interviews with volunteers about how they find out about these experiments and what are their key motivations in taking part in experiments.

An affinity map was coded to find key design opportunities.

Use Journey Map to identify problematic processs in recruiting flow for both parties

I also created a journey map to specify the problem areas and help me understand the whole user journey for both researchers and their targetd experiment participants.


IDEATIONS

Imaging a product that connects these two type of users

  • What platform is more accessible to both type of users?
  • What features will solve users painpoints?
  • How do we structure these features in end-to-end user workflows?

Imaging New Workflows

I created new workflows that:

  • Researchers can utilize automatic information exchange and updates, so that they can save time on preparing for actual experiment.
  • Researchers can access past Experiment Participants data, so that they can reach out for future opportunities.
  • Experiment Participants can get reminder as well as reschedule, so that they can show up to the experiment on time.
  • Experiment Participants can check in and get rewards online, so that they get rewards while the system updates there status data for Researchers.

MOBILE APP: EXPERIMENT PARTICIPANTS

Create Participants Workflows

With existing ideation, I created user flow chart to connect the fragmented wireframes. Also, I add in new wireframes with existing ones to envision a more structured, well-rounded system for users.

Improvements 1: Make the availabe time slots more discoverable

I conducted peer review with the prototypes with classmats from U of M. Based on their feedbacks, I promote the account sign up, experiment sign up and check in prototypes.

The first feedback is that users find the calendar selection is not clear which dates are available to them, and for them that is one of the most important information.

Solution:
1. Add confirmation page with "add to calendar" button.
2. Modify time slot interface.

Improvements 2: Educate user to use the checkin functions by showing detailed status

User are not familiar with "checking in", therefore they need more instructions and explainations while using this feature.

Solution:
1. Add description of distance limits and confirmation page for check in page.
2.Show loading animations with status and available user actions.

WEB APP: RESEARCHERS

Version 1: Testing dashboard concepts with Researchers

In the first design, I explored ideas of creating a dashboard, and while gathering feedbacks, I found several problems of this design.

Users could not finish all the tasks in the workflow using these interfaces:

  • 1. Post New Experiment : Current posting content is not consistent with what volunteers can see in their mobile app.
  • 2. Recruitment Process Tracking: Charts don’t make sense and not interactive.
  • 3. Appointments Tracking: Confusing layouts and interactions of appointment section.
  • 4. Volunteer Long-term Tracking: Lack tools for managing volunteers.
  • 5. Give Reward and Messages: No such option.

There are also a couple of interaction design problem I found:

  • 6. Scrollable: The page is not scrollable, which failed to utilize the strength of a website.
  • 7. Dashboard: Dashboard is badly organized without highlighting any important content.
  • 8. Navigation: The navigation and layout across pages are not consistent.

Version 2: Researchers don't need dashboard, they care about EFFICIENCY and need to know "What's Next"

Based on these feedbacks and insights, I was able to answer these three questions:

1. Is the user trying to just monitor KPIs, across the organization over an indefinite period?
No. According to the user research, they care about EFFICIENCY instead of drawing insights of recruiting data. The only data they care about recruiting volunteers is whether they reached their expecting number, which is PROGRESS. ​

2. What are their potential behaviors?
Tracking progress and look for time of appointments. ​

3. Do we really need so many charts in dashboard?
For tracking part, they don't need sophisticated dashboard. What they really need to increase efficiency is a way to show them "What's Next".
